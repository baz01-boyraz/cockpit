When cutting a release, `git push --follow-tags` only pushes annotated tags. A lightweight `vX.Y.Z` tag is NOT pushed by it, so the GitHub Actions Release workflow (which triggers on the tag) never runs. Fix: push the tag explicitly (`git push origin vX.Y.Z`) — hit during the v0.1.30 release. Combine with the existing rule: bump version first, then tag after the bump.
