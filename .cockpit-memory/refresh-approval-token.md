The app:refresh lifecycle script must check for a short-lived, single-use Cockpit UI approval token before calling app.quit(). Without a valid token, the script exits before closing the app. The token is generated by the Cockpit UI and bound to a specific checkout and operation. This is enforced both by the shell script gate and by native Claude hook / Codex command rules, which forbid app:refresh unless the explicit consent + UI token are both present. Test, build, typecheck, and screenshot actions never count as refresh consent.
